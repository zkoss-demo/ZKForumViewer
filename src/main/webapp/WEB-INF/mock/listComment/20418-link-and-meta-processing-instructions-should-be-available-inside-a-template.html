<?link?> and <?meta?> processing instructions should be available inside a template

stevesobol
23 Aug 2012 04:50:35 GMT
23 Aug 2012 04:50:35 GMT

Hello,

I'm using templates wherever possible in my ZK projects.

I have a page, let's call it sample.zul. It pulls in a template called main.zul:

<?init class="org.zkoss.zk.ui.util.Composition" arg0="/WEB-INF/templates/main.zul"?>

main.zul basically contains the entire page structure, with only the page-specific content residing in sample.zul.

Given these facts, it'd be really nice if I could use certain processing instructions within the template. ESPECIALLY <? link ?> - because right now that PI must be used in sample.zul, and if I create other pages that use the same stylesheets I must duplicate the <? link ?> PI on each page.

I feel the same way about <? meta ?>.

It'd really be helpful if I was able to use those PI's within a template/

Any chance we could make that happen?
