I have 2 zul files.
One uses <include> to include the other.

both have window node which defines a VM.

When I use a custom parameter with a simple string it will work.
If I use the @load annotation to pass in an object from the (outer) VM then this property is evaluated put when the value is assigned to the parameter this can not be found.

I just tested this with the newest 6.0.2 build after it fails with 6.0.1

The main zul page has:

<window width="100%"
   apply="org.zkoss.bind.BindComposer" viewModel="@id('vm') @init('org.XXX.web.controller.BenutzerVM')" >

 <include id="RollenListe" src="/zk/genericDualListbox.zul"  kind="Rollen"  list2="@load(vm.test)"  />
...

public class GenericDualListSelectionVM{

 @Init

   }
....
